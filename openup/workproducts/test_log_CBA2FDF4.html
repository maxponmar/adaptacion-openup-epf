Artifact: Registro de Pruebas
Registra de manera sistemática y detallada la información relevante sobre las pruebas realizadas en el software durante el proceso de desarrollo y pruebas. Contiene registros de resultados de pruebas, evidencia de errores y detalles de las actividades de prueba.
Domains: Test
Work Product Kinds: Project Data
Purpose

Proporcionar un rastreo completo de las actividades de prueba y sus resultados. Sirve como un registro centralizado de las pruebas ejecutadas, lo que facilita la identificación y resolución de problemas, la toma de decisiones informadas y la verificación de la calidad del software.

Description
Main Description

El Test Log incluye información detallada sobre las actividades de prueba:

  • Resumen de Pruebas: Proporciona un resumen general de las pruebas realizadas, incluyendo la fecha de inicio y finalización de la prueba, el alcance de las pruebas y los objetivos.
  • Casos de Prueba: Enumera los casos de prueba específicos que se ejecutaron durante las pruebas. Cada caso de prueba se describe detalladamente, incluyendo su propósito, los pasos a seguir, los datos de entrada y los resultados esperados.
  • Resultados de Pruebas: Registra los resultados de cada caso de prueba, indicando si se pasaron o fallaron. También se pueden incluir detalles adicionales sobre el comportamiento observado durante las pruebas.
  • Evidencia de Errores: Si se detectaron errores durante las pruebas, el Test Log registra información detallada sobre cada error, incluyendo una descripción, una gravedad, una fecha de descubrimiento y el estado de resolución.
  • Ambiente de Pruebas: Describe el entorno de pruebas utilizado, incluyendo la configuración de hardware y software, las herramientas de prueba y cualquier configuración específica necesaria para reproducir las pruebas.
  • Responsables de Pruebas: Indica quiénes llevaron a cabo las pruebas, incluyendo los nombres de los miembros del equipo o las personas responsables de la ejecución de las pruebas.
  • Observaciones y Comentarios: Permite a los probadores incluir observaciones adicionales, comentarios o notas relevantes sobre las pruebas realizadas.
Brief OutlineEl Test Log es un artefacto crítico en el proceso de aseguramiento de la calidad del software. Facilita la trazabilidad de las pruebas realizadas, ayuda en la identificación temprana de problemas y contribuye a la mejora continua de la calidad del software a lo largo del desarrollo.
Key Considerations
  • El Test Log debe mantenerse actualizado a medida que se realizan nuevas pruebas y se obtienen resultados adicionales.
  • La consistencia en la documentación y la notificación temprana de errores son fundamentales para garantizar una resolución eficiente de problemas.
Tailoring
Representation Options

Since this is a collection of raw data for subsequent analysis, it can be represented in a number of ways.

Some suggestions are:

  • For manual tests, log the actual results on a copy of the manual Test Script
  • For automated tests, direct the output to log files that you can trace back to the automated Test Script
  • Track raw results data in a test management tool.
