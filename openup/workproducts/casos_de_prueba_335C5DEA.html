Artifact: Casos de Prueba
Este artefacto es la especificación de un conjunto de entradas de prueba, condiciones de ejecución y resultados esperados, identificados con el propósito de evaluar algún aspecto particular de un escenario.
Domains: Test
Work Product Kinds: Specification
Purpose

El propósito de este artefacto es:

- Proporcionar una forma de capturar las entradas de prueba, las condiciones y los resultados esperados para un sistema.
- Identificar sistemáticamente los aspectos del software que se deben probar.
- Especificar si se ha alcanzado un resultado esperado basado en la verificación de un requisito del sistema, un conjunto de requisitos o un escenario.

Description
Main Description

Un caso de prueba especifica las condiciones que deben validarse para permitir una evaluación de aspectos particulares del sistema bajo prueba. Un caso de prueba es más formal que una idea de prueba y generalmente toma la forma de una especificación. En entornos menos formales, los casos de prueba se pueden crear identificando un ID único, un nombre, datos de prueba asociados y resultados esperados. Para obtener un ejemplo de este tipo de caso de prueba, consulte la Plantilla: Caso de Prueba.

Los casos de prueba pueden derivarse de muchas fuentes, pero generalmente incluirán un subconjunto de los requisitos (como casos de uso, características de rendimiento, consideraciones de confiabilidad) y otros tipos de atributos de calidad. Para obtener más información sobre los tipos de pruebas y su relación con los atributos de calidad de las pruebas, consulte el Concepto: Tipos de Pruebas.



Evaluación de una Sección de Tesis 

Objetivo del Test: Verificar que la aplicación evalúa correctamente una sección de la tesis y proporciona la retroalimentación adecuada.

Precondiciones:

  • El usuario debe haber iniciado sesión en la aplicación.
    El usuario debe tener una tesis cargada en la aplicación con al menos una sección para evaluar.

Pasos:

  1. El usuario navega a la sección "Evaluación de Tesis".
  2. El usuario selecciona la sección de la tesis que desea evaluar.
  3. El usuario solicita la evaluación de la sección.
  4. La aplicación procesa la solicitud y muestra los resultados de la evaluación, incluyendo la puntuación y las recomendaciones.

Criterios de Aceptación:

  • La aplicación debe mostrar los resultados de la evaluación en un tiempo razonable.
  • La puntuación y las recomendaciones deben ser coherentes con el contenido de la sección evaluada.
