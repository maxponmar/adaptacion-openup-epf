Artifact: Caso de Uso
Es un artefacto utilizado en el análisis y diseño de sistemas de software para describir una interacción específica entre el sistema y sus usuarios (actores). Representa una unidad funcional que describe cómo un actor interactúa con el sistema para lograr un objetivo particular.
Domains: Requirements
Work Product Kinds: Model Element, Specification
Purpose

El propósito de un Caso de Uso es capturar y documentar el comportamiento funcional del sistema desde la perspectiva del usuario. Proporciona una representación clara y comprensible de las interacciones que ocurren en el sistema y ayuda a establecer los requisitos funcionales.

Description
Main Description

Un Caso de Uso comprende los siguientes aspectos clave:

  • Nombre del Caso de Uso: Un nombre descriptivo que identifica el objetivo o la función que se describe, por ejemplo, "Registrar Usuario" o "Realizar Pago".
  • Actores: Los actores son las personas, sistemas externos o entidades que interactúan con el sistema. Se identifican y se describen los actores involucrados en el Caso de Uso.
  • Descripción: Una breve descripción que proporciona una visión general de la interacción entre el actor y el sistema.
  • Flujo Principal: El flujo principal describe las acciones secuenciales que ocurren cuando se realiza el Caso de Uso. Es una descripción paso a paso de cómo se logra el objetivo del Caso de Uso.
  • Escenarios Alternativos: Los escenarios alternativos representan las posibles ramificaciones o variaciones en el flujo principal debido a condiciones excepcionales o decisiones tomadas por el usuario o el sistema.
  • Precondiciones: Las precondiciones son las condiciones que deben cumplirse antes de que el Caso de Uso pueda ejecutarse correctamente.
  • Postcondiciones: Las postcondiciones son las condiciones que se cumplen después de que se haya ejecutado el Caso de Uso.
  • Diagrama de Casos de Uso: A menudo, se representa gráficamente el Caso de Uso en un diagrama de Casos de Uso para mostrar las relaciones entre los actores y los casos de uso.
Brief OutlineLos Casos de Uso son una parte fundamental del análisis de requisitos y el diseño de sistemas de software. Proporcionan una forma estructurada y comprensible de capturar y comunicar cómo interactúan los usuarios con el sistema y qué funcionalidades se espera que proporcione.

Key Considerations
  • Los Casos de Uso son una herramienta efectiva para comunicar y documentar los requisitos funcionales del sistema.
  • Pueden ser utilizados para validar los requisitos con los Stakeholders y para guiar el diseño y desarrollo del sistema.
Tailoring
Representation Options

Decide the extent to which you will elaborate on Use Cases:

  • Describe only major flows?
  • Describe only the most important Use Cases?
  • Fully describe preconditions and post-conditions?
  • Describe scenarios first, and then raise the level of abstraction by describing Use Case flows?

Some projects apply Use Cases informally to help discover requirements, documenting and maintaining these requirements in another form such as user stories. How you tailor Use Cases may depend on project size, team experience, your tool set, the customer relationship, and so forth. See Guideline: Detail Use Cases and Scenarios for guidance related to documenting Use Cases.
