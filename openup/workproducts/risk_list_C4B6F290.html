Artifact: Lista de Riesgos
Es un registro que identifica y evalúa los posibles riesgos y amenazas que podrían afectar al proyecto de tesis relacionado con la plataforma de evaluación de contenido de tesis universitarias. Esta lista sirve como un instrumento para la gestión proactiva de riesgos.
Domains: Project Management
Work Product Kinds: Project Data
Purpose

El propósito de este artefacto es anticipar y abordar posibles desafíos y problemas que podrían surgir durante el curso del proyecto. Facilita la identificación, evaluación y mitigación de riesgos, lo que ayuda a minimizar posibles impactos negativos en el proyecto.

Description
Main Description

La Lista de Riesgos es un artefacto que involucra los siguientes elementos clave:

  • Identificación de Riesgos: Enumera los riesgos específicos que podrían surgir durante el proyecto. Esto incluye eventos inciertos que podrían tener un impacto negativo en el alcance, los plazos, los costos o la calidad del proyecto.
  • Evaluación de Riesgos: Para cada riesgo identificado, se realiza una evaluación que incluye la probabilidad de que ocurra y el impacto potencial en el proyecto si el riesgo se materializa. Esto ayuda a priorizar los riesgos en función de su gravedad.
  • Estrategias de Mitigación: Para los riesgos evaluados, se desarrollan estrategias de mitigación que describen cómo se pueden prevenir, reducir o abordar los riesgos. Esto incluye medidas preventivas y planes de contingencia.
  • Responsables: Se asigna a individuos o equipos responsables de supervisar y ejecutar las estrategias de mitigación para cada riesgo. Esto asegura que se tomen acciones adecuadas para reducir los riesgos.
  • Seguimiento y Actualización: La Lista de Riesgos se mantiene actualizada a lo largo del proyecto. Se registran cambios en la probabilidad o el impacto de los riesgos, y se ajustan las estrategias de mitigación según sea necesario.
Brief OutlinePermite una identificación temprana y una gestión eficiente de posibles desafíos. Ayuda a mantener el proyecto en el camino correcto y a minimizar los impactos negativos en caso de que ocurran riesgos.

Key Considerations
  • La gestión de riesgos es un proceso continuo y proactivo que debe llevarse a cabo a lo largo de todo el proyecto.
  • La comunicación efectiva sobre los riesgos con el equipo de proyecto y los stakeholders es esencial para garantizar una respuesta rápida y eficaz.
Tailoring
Representation Options

Option: list of risks captured in the project plan

In this approach you put the overall risk list in the project plan. The iteration plan will contain only the tasks you will be doing during the iteration to mitigate the risks. This will ensure that the iteration plan contains only iteration information. The project plan has to be revisited constantly as you update risks.
