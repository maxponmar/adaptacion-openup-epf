Artifact: Work Items List
Es un registro detallado de todas las tareas, actividades y entregables específicos que deben realizarse a lo largo del proyecto de tesis relacionado con la plataforma de evaluación de contenido de tesis universitarias. Esta lista se utiliza para gestionar y dar seguimiento a todas las actividades del proyecto.
Domains: Project Management
Work Product Kinds: Project Data
Purpose

Proporcionar una estructura organizativa y una visión integral de todas las tareas que componen el proyecto. Facilita la planificación, la asignación de recursos, el seguimiento del progreso y la gestión eficaz de las actividades del proyecto.

Description
Main Description

La Lista de Elementos de Trabajo es un artefacto fundamental para el control y la gestión del proyecto de tesis y la plataforma de evaluación de tesis universitarias. Incluye lo siguiente:

  • Tareas y Actividades: Enumera todas las tareas y actividades específicas que deben llevarse a cabo en el proyecto, desde la investigación inicial hasta el desarrollo de la plataforma y las actividades de evaluación.
  • Entregables: Identifica los entregables clave del proyecto, como documentos, informes, prototipos, módulos de software y otros resultados tangibles que deben ser producidos.
  • Asignación de Responsabilidades: Especifica quién es responsable de cada tarea o actividad. Esto asegura que cada tarea tenga un propietario designado.
  • Plazos y Fechas Límite: Establece las fechas de inicio y finalización previstas para cada tarea y actividad. Esto ayuda a garantizar que el proyecto se mantenga en el camino correcto.
  • Prioridades: Puede asignar prioridades a las tareas para indicar su importancia relativa y ayudar en la gestión de recursos.


Investigación Inicial:

  • Realizar una revisión de literatura para comprender los conceptos clave relacionados con el procesamiento de lenguaje natural, la inteligencia artificial y la evaluación de tesis.
  • Investigar las herramientas y tecnologías disponibles, como Freeling y bases de datos, que pueden ser útiles en el proyecto.

Definición de Requisitos:

  • Identificar y documentar los requisitos funcionales y no funcionales de la plataforma.
  • Realizar entrevistas con potenciales usuarios para comprender sus necesidades y expectativas.

Diseño de la Arquitectura:

  • Crear una arquitectura de alto nivel para la plataforma, definiendo componentes principales y sus interacciones.
  • Seleccionar las tecnologías y herramientas adecuadas para la implementación.

Desarrollo de la Interfaz de Usuario (UI):

  • Diseñar la interfaz de usuario de la plataforma, incluyendo la página de inicio de sesión y las áreas de evaluación de tesis.
  • Implementar la UI de manera que sea fácil de usar y atractiva visualmente.

Desarrollo de la Funcionalidad Principal:

  • Desarrollar los módulos de procesamiento de lenguaje natural e inteligencia artificial para la evaluación de tesis.
  • Integrar la herramienta de Freeling para el análisis lingüístico.

Implementación de la Base de Datos:

  • Diseñar y crear la estructura de la base de datos para almacenar tesis, resultados de evaluación y datos de usuario.
  • Implementar la funcionalidad de guardar y recuperar resultados.

Desarrollo del Sistema de Inicio de Sesión:

  • Crear un sistema de inicio de sesión seguro y autenticación de usuarios.
  • Gestionar perfiles de usuario y roles.

Implementación de Almacenamiento de Resultados:

  • Desarrollar la capacidad de guardar y consultar resultados de evaluación de tesis para los usuarios registrados.

Pruebas y Depuración:

  • Realizar pruebas exhaustivas de todas las funcionalidades para identificar y corregir errores.
  • Verificar que el procesamiento de lenguaje natural e inteligencia artificial produzca resultados precisos.

Capacitación del Modelo de IA:

  • Entrenar y ajustar el modelo de inteligencia artificial para mejorar la precisión de la evaluación de tesis.

Pruebas de Usuario:

  • Realizar pruebas con usuarios reales para obtener retroalimentación sobre la usabilidad y la calidad de las evaluaciones.

Documentación:

  • Crear documentación técnica y de usuario para la plataforma, incluyendo manuales y tutoriales.

Despliegue:

  • Implementar la plataforma en un entorno de producción y configurar la infraestructura necesaria, como servidores y bases de datos.

Monitoreo y Mantenimiento:

  • Establecer un sistema de monitoreo continuo para asegurar el rendimiento y la disponibilidad de la plataforma.
  • Realizar mantenimiento regular y actualizaciones según sea necesario.

Pruebas de Seguridad:

  • Realizar pruebas de seguridad para identificar y corregir posibles vulnerabilidades.

Evaluación de Resultados:

  • Evaluar el rendimiento de la plataforma en términos de precisión en la evaluación de tesis y satisfacción del usuario.

Entrega y Presentación de Tesis:

  • Permitir a los usuarios cargar sus tesis, recibir evaluaciones y utilizar los resultados para mejorar sus trabajos académicos.

Soporte y Atención al Cliente:

  • Proporcionar soporte técnico y atención al cliente a los usuarios de la plataforma.

Evaluación Final del Proyecto:

  • Realizar una evaluación integral del proyecto para identificar lecciones aprendidas y áreas de mejora.

Work Items should contain estimates, see Guideline: Managing Work Items and Guideline: Agile Estimation.

Key Considerations
  • La Lista de Elementos de Trabajo debe mantenerse actualizada a medida que evoluciona el proyecto y se identifican nuevas tareas o cambios en las fechas límite.
  • Es importante realizar un seguimiento constante del progreso y ajustar la lista según sea necesario para abordar los cambios en el alcance o las prioridades.
