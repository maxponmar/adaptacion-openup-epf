Description
Main Description
  • Actores: Usuario no registrado, Sistema de Registro.
  • Descripción: Este caso de uso describe cómo un usuario no registrado puede registrarse en la aplicación.

Flujo Principal:

  1. El Usuario no registrado accede a la página de registro.
  2. El Sistema de Registro muestra el formulario de registro.
  3. El Usuario completa el formulario con su información personal.
  4. El Sistema de Registro verifica la validez de los datos ingresados.
  5. Si los datos son válidos, el Sistema de Registro crea una cuenta para el Usuario.
  6. El Usuario recibe una confirmación de registro.

Escenarios Alternativos:

  • Si los datos ingresados son inválidos, el Sistema de Registro muestra mensajes de error y solicita que se corrijan.
  • Si el Usuario ya tiene una cuenta, se muestra un mensaje indicando que ya está registrado.
  • Precondiciones: El Usuario no debe estar registrado en la aplicación.
  • Postcondiciones: El Usuario queda registrado en la aplicación.
  • Diagrama de Caso de Uso: En un diagrama de casos de uso, este caso de uso estaría conectado al actor "Usuario no registrado".
